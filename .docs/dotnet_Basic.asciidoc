== Objetivo do nível

Este arquivo cobre fundamentos que um desenvolvedor iniciante em .NET precisa explicar com segurança: plataforma, runtime, tipos, coleções, exceções, LINQ e assincronismo básico.

== Plataforma e runtime

.Qual é a diferença entre C# e .NET?
[%collapsible]
====
C# é a linguagem. .NET é a plataforma de execução e bibliotecas.

* C# define sintaxe, tipos, expressões, classes, interfaces e recursos da linguagem.
* .NET fornece runtime, biblioteca base, SDK, ferramentas de build, gerenciamento de memória e suporte a vários tipos de aplicação.
* Um programa C# normalmente é compilado para IL e executado pelo runtime .NET.
====

.O que é o SDK do .NET?
[%collapsible]
====
O SDK é o conjunto de ferramentas para criar, compilar, testar, empacotar e executar projetos .NET.

Ele inclui a CLI `dotnet`, compiladores, templates, MSBuild, runtime de desenvolvimento e tarefas de restauração de pacotes.
Em um repositório, o arquivo `global.json` pode fixar a versão esperada do SDK.
====

.O que é o runtime do .NET?
[%collapsible]
====
O runtime executa aplicações .NET.

Ele carrega assemblies, executa código gerenciado, aplica segurança de tipos, gerencia exceções, coordena threads e executa o garbage collector.
O SDK é usado para desenvolver; o runtime é necessário para executar.
====

.O que é CLR?
[%collapsible]
====
CLR significa Common Language Runtime.

Ele é o ambiente de execução do .NET: carrega código, executa IL compilado para código nativo, gerencia memória, exceções, interoperabilidade e segurança de tipos.
No .NET moderno, o termo aparece como parte da arquitetura do runtime.
====

.O que é IL?
[%collapsible]
====
IL, ou Intermediate Language, é a representação intermediária gerada pelo compilador C#.

O código C# não vira diretamente código de máquina na compilação normal.
Ele vira IL em assemblies; depois o runtime compila o IL para código nativo quando necessário.
====

.O que é JIT?
[%collapsible]
====
JIT significa Just-In-Time compilation.

É o processo em que o runtime converte IL em código nativo durante a execução.
Isso permite portabilidade entre arquiteturas e otimizações feitas conforme o ambiente real de execução.
====

.O que é um assembly?
[%collapsible]
====
Um assembly é uma unidade de implantação e versionamento no .NET.

Normalmente aparece como `.dll` ou `.exe` e contém IL, metadados, referências e informações de versão.
Projetos C# compilados geram assemblies.
====

.Qual é a diferença entre projeto e solução?
[%collapsible]
====
Um projeto descreve como compilar uma unidade de código, geralmente por meio de um arquivo `.csproj`.

Uma solução agrupa vários projetos para build, execução e organização.
Neste repositório, a solução raiz é `CSharpCodePortfolio.slnx`.
====

.O que é NuGet?
[%collapsible]
====
NuGet é o gerenciador de pacotes usado pelo ecossistema .NET.

Ele permite referenciar bibliotecas externas, restaurar versões específicas e distribuir pacotes.
As referências ficam normalmente em arquivos `.csproj`.
====

.O que significa código gerenciado?
[%collapsible]
====
Código gerenciado é executado sob controle do runtime .NET.

Ele recebe serviços como garbage collection, verificação de tipos, exceções, metadados e interoperabilidade controlada.
Código não gerenciado é executado diretamente fora desse modelo, como código nativo C/C++.
====

== Tipos e memória

.Qual é a diferença entre value type e reference type?
[%collapsible]
====
Value types representam o valor diretamente; reference types armazenam uma referência para um objeto.

* Exemplos de value types: `int`, `bool`, `DateTime`, `decimal`, `struct`.
* Exemplos de reference types: `class`, `string`, arrays, delegates e interfaces em tempo de execução.
* A diferença afeta cópia, igualdade, nulidade e alocação.
====

.O que são stack e heap em termos práticos?
[%collapsible]
====
Stack é usada para frames de chamadas, variáveis locais e controle de execução.
Heap é usada para objetos gerenciados que precisam sobreviver além do escopo imediato.

Essa explicação é uma simplificação: o compilador e o runtime podem otimizar alocações.
O ponto essencial é entender tempo de vida, referência e custo de alocação.
====

.O que é boxing?
[%collapsible]
====
Boxing é converter um value type em `object` ou em uma interface implementada por ele.

Isso normalmente cria uma alocação no heap.
Exemplo: `object valor = 42;`.
====

.O que é unboxing?
[%collapsible]
====
Unboxing é recuperar o value type de dentro de um objeto boxed.

Exemplo: `int numero = (int)valor;`.
O tipo precisa ser compatível; caso contrário, ocorre exceção em tempo de execução.
====

.O que é nullable reference type?
[%collapsible]
====
Nullable reference types são anotações e análises do compilador para reduzir erros de `null`.

Com `Nullable` habilitado, `string` significa referência esperada como não nula, enquanto `string?` permite nulo.
O recurso ajuda no design de contratos e evita muitas falhas por `NullReferenceException`.
====

.Qual é a diferença entre `var` e `dynamic`?
[%collapsible]
====
`var` é tipagem estática com inferência em tempo de compilação.
`dynamic` adia a resolução de membros para tempo de execução.

Use `var` quando o tipo for evidente pelo lado direito.
Use `dynamic` apenas em cenários específicos, como interoperabilidade dinâmica.
====

.O que são `const` e `readonly`?
[%collapsible]
====
`const` define um valor constante em tempo de compilação.
`readonly` define um campo que só pode ser atribuído na declaração ou no construtor.

Use `const` para valores realmente imutáveis e universais.
Use `readonly` para valores definidos em tempo de construção.
====

.O que é uma propriedade em C#?
[%collapsible]
====
Uma propriedade expõe acesso controlado a dados de um objeto.

Ela pode encapsular validação, cálculo ou armazenamento interno.
Exemplo: `public string Nome { get; init; } = "";`.
====

.Qual é a diferença entre campo e propriedade?
[%collapsible]
====
Campo é armazenamento direto dentro do tipo.
Propriedade é uma API pública ou interna para acesso a dados.

Como regra, exponha propriedades e mantenha campos como detalhe de implementação.
====

.O que é `string` em C#?
[%collapsible]
====
`string` é um tipo de referência imutável que representa texto Unicode.

Operações que parecem modificar uma string normalmente criam outra instância.
Para muitas concatenações em loop, `StringBuilder` pode ser mais adequado.
====

== Fluxo, coleções e LINQ

.Qual é a diferença entre array e `List<T>`?
[%collapsible]
====
Array tem tamanho fixo depois de criado.
`List<T>` é uma coleção genérica redimensionável.

Use arrays quando o tamanho é fixo ou quando uma API exige array.
Use `List<T>` para coleções que crescem ou encolhem durante o fluxo.
====

.Por que preferir coleções genéricas?
[%collapsible]
====
Coleções genéricas como `List<T>` e `Dictionary<TKey,TValue>` preservam tipo em tempo de compilação.

Elas evitam casts desnecessários, reduzem boxing e tornam o código mais claro.
`ArrayList` existe por compatibilidade, mas raramente é a escolha correta em código moderno.
====

.O que é `Dictionary<TKey,TValue>`?
[%collapsible]
====
É uma coleção de chave e valor otimizada para busca por chave.

Use quando você precisa localizar valores rapidamente por identificador.
A chave deve ter igualdade e hash coerentes.
====

.O que é LINQ?
[%collapsible]
====
LINQ é um conjunto de recursos para consultar dados em C#.

Ele permite filtrar, projetar, ordenar, agrupar e transformar coleções com métodos como `Where`, `Select`, `OrderBy` e `GroupBy`.
====

.Qual é a diferença entre `IEnumerable<T>` e `List<T>`?
[%collapsible]
====
`IEnumerable<T>` representa uma sequência que pode ser enumerada.
`List<T>` é uma coleção concreta em memória.

Use `IEnumerable<T>` em APIs que só precisam percorrer dados.
Use `List<T>` quando precisa adicionar, remover, indexar ou materializar resultados.
====

.O que é deferred execution no LINQ?
[%collapsible]
====
Deferred execution significa que a consulta só executa quando é enumerada.

Exemplo: uma chamada a `Where` monta a consulta; `ToList`, `foreach` ou `Count` executam.
Esse comportamento é útil, mas pode causar surpresas se a fonte mudar antes da enumeração.
====

.O que é uma exceção?
[%collapsible]
====
Exceção é um mecanismo para sinalizar erro ou condição inesperada durante a execução.

Use exceções para falhas reais, não para fluxo normal.
Capture exceções quando puder recuperar, adicionar contexto ou converter para uma falha mais adequada.
====

.Qual é a diferença entre `throw` e `throw ex`?
[%collapsible]
====
`throw` preserva o stack trace original dentro de um `catch`.
`throw ex` reinicia parte do stack trace e dificulta diagnóstico.

Ao relançar a mesma exceção, prefira `throw;`.
====

.O que é `using` statement?
[%collapsible]
====
`using` garante que `Dispose` seja chamado ao final do escopo.

Ele é usado com tipos que implementam `IDisposable`, como streams, conexões e outros recursos que exigem liberação determinística.
====

.O que é `async` e `await`?
[%collapsible]
====
`async` marca um método que pode aguardar operações assíncronas.
`await` libera o fluxo enquanto uma operação assíncrona não termina.

Esse modelo é útil para I/O, rede, banco de dados e arquivos, sem bloquear a thread durante a espera.
====

.Qual é a diferença entre `Task` e `Thread`?
[%collapsible]
====
`Thread` representa uma thread do sistema operacional.
`Task` representa uma unidade de trabalho assíncrona ou paralela no modelo TPL.

Na maioria dos códigos modernos, use `Task` e APIs assíncronas em vez de gerenciar threads manualmente.
====
